# 個人記帳簿 Web 應用程式 - 產品需求文件 (PRD)

**專案名稱：** 個人記帳簿 (Personal Finance Tracker)
**專案描述：** 一個直覺且簡單的 Web 應用程式，幫助使用者輕鬆記錄每日的收入與支出，並透過圖表與列表清楚掌握個人的財務狀況與餘額變化。

---

## 1. 專案概述

### 背景與動機
許多人在日常生活中缺乏記錄個人財務的習慣，或是覺得市面上的記帳軟體功能過於複雜。為了幫助使用者能更輕鬆地掌握自己的金錢流向與餘額，我們需要一個專注於核心記帳功能、介面清晰簡單的輕量級記帳 Web 應用程式。

### 目標用戶
- 有個人記帳需求、希望掌握每日收支狀況的一般大眾
- 覺得市面上的記帳軟體過於複雜，只需要基本收支記錄功能的使用者
- 學生或上班族想要輕鬆掌握生活費餘額

### 核心價值主張
- **簡單直覺**：去掉複雜的繁瑣設定，專注於記錄「收入」與「支出」。
- **一目了然**：透過快速顯示目前的總餘額與最近紀錄，讓使用者掌握財務現況。

---

## 2. 功能需求

### 2.1 新增收支紀錄
- 作為 **使用者**，我希望 **能夠透過表單輸入金額、類別、日期與備註**，以便 **記錄每一筆收入或支出**。

### 2.2 檢視總餘額與統計
- 作為 **使用者**，我希望 **能在首頁看到目前的總餘額、本月總收入與總支出**，以便 **快速掌握最近的財務狀況**。

### 2.3 檢視歷史紀錄
- 作為 **使用者**，我希望 **能夠看到依照時間排序的收支紀錄列表**，以便 **回顧過去的消費明細**。

### 2.4 刪除紀錄
- 作為 **使用者**，我希望 **能夠刪除輸入錯誤的收支紀錄**，以便 **維持帳目的正確性**。

### 2.5 紀錄篩選與分類
- 作為 **使用者**，我希望 **能根據「收入/支出」或「類別」來篩選紀錄**，以便 **分析特定的花費項目（例如：餐飲費）**。

---

## 3. 非功能需求

### 技術限制
- 後端框架：使用 Python 的 **Flask**
- 模板引擎：使用 **Jinja2** 進行前端畫面渲染
- 資料庫：使用 **SQLite** 進行資料儲存，結構簡單且易於部署
- 前端技術：HTML, CSS (適當加入 vanilla JS 以提升基本互動體驗，如表單驗證)

### 效能與安全考量
- **效能**：應用程式應具備輕量化特性，首頁載入時間與新增紀錄的回應時間需在 1 秒內。
- **安全**：
  - 需防範基本的 SQL Injection 攻擊（透過 SQLAlchemy 或參數化查詢實作）。
  - 對表單輸入進行基礎驗證（例如金額必須大於 0、日期不可為空等）。

---

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必要功能 - 第一階段必須上線)
- 新增收入或支出紀錄（包含：種類、金額、日期、備註）。
- 儀表板顯示當前「總餘額」。
- 條列式顯示最近所有的收支明細列表。

### Should Have (重要功能 - 建議有)
- 提供不同常見的分類標籤（如：食、衣、住、行、薪水等）供使用者選擇。
- 可以刪除或編輯既有的一筆紀錄。
- 顯示當月總收入與總支出統計。

### Nice to Have (加分功能 - 看開發時程決定)
- 按月或按類別的收支圖表（圓餅圖或長條圖）呈現。
- 資料匯出功能（如匯出成 CSV 格式）。
- 關鍵字搜尋歷月紀錄。
