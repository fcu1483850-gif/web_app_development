# 產品需求文件 (PRD)

專案名稱：校園美食推薦平台
專案描述：一個專為大學生設計的推薦平台，具備地圖標註、搜尋與口味評分功能，幫助學生解決不知道吃什麼的煩惱，並找到校園周邊真實的高評價美食。

## 1. 專案概述
- **背景與動機**：大學生（特別是新生）經常面臨「不知道吃什麼」的困難查。傳統地圖工具資訊過於廣泛，缺乏專注於校園周邊且具真實學生口味評分的平台。
- **目標用戶**：大學生、教職員，以及對校園周邊美食感興趣的訪客。
- **核心價值主張**：提供一個操作簡單直覺的平台，結合地圖標註與評分機制，讓用戶能快速找到符合口味的校園美食。

## 2. 功能需求
1. **美食地圖標註 (Map Marking)**
   - 作為使用者，我希望能在地圖上直覺地看到周邊餐廳的地標，以便判斷餐廳的相對位置與遠近距離。
2. **餐廳搜尋 (Search)**
   - 作為使用者，我希望能透過關鍵字搜尋餐廳名稱或食物分類，以便快速找到我想吃的餐點。
3. **口味評分與評論 (Taste Ratings)**
   - 作為使用者，我希望能對體驗過的餐廳給予口味星級評分並留下簡短文字評論，以便將真實心得分享給其他人。
4. **餐廳詳細資訊檢視**
   - 作為使用者，我希望點擊地圖標記或搜尋結果後，能看到詳細資訊（如地址、營業時間、招牌介紹與平均評分），以便決定是否前往用餐。
5. **餐廳資料新增/編輯**
   - 作為管理員或貢獻者，我希望能新增未收錄的餐廳或編輯舊有資訊，以便讓平台資料保持最新與完整。

## 3. 非功能需求
- **技術限制**：
  - 後端：Flask (Python)
  - 前端：HTML + Vanilla CSS + JavaScript，並搭配 Jinja2 模板渲染。
  - 資料庫：SQLite (適合 Prototype 及單機/小型應用)。
- **效能與安全考量**：
  - 由於是初期 Prototype，頁面反應需快速順暢。
  - 需防範基本的 XSS 攻擊（針對評論與輸入框進行字元過濾或跳脫處理）。
  - 防止 SQL Injection，建議搭配 ORM（如 SQLAlchemy）或使用參數化查詢。

## 4. MVP 範圍
- **Must Have (必須有)**：
  - 首頁地圖展示與餐廳基礎標記。
  - 關鍵字搜尋。
  - 簡單的星級口味評分及留言功能。
  - 餐廳詳細資料頁。
- **Should Have (應該有)**：
  - 餐廳分類標籤（例如：中式、西式、早午餐、宵夜）。
  - 基本防呆邏輯（限制空白評論或錯誤評分輸入）。
- **Nice to Have (加分項)**：
  - 使用者註冊與登入功能（識別是誰發表的評論）。
  - 收藏喜愛餐廳清單。
  - 上傳餐點照片。
